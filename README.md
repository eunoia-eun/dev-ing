# 사업장 직원 건강관리 시스템

보건관리자(간호사)를 위한 임직원 건강관리 웹앱입니다.
**레이어드/클린 아키텍처 + 테스트 하네스(Harness Engineering)** 구조로 작성되었습니다.

## 주요 기능

| 메뉴 | 설명 |
| --- | --- |
| 🏠 **대시보드** | 검진 도래·방문·재고·프로그램 현황을 한눈에 요약 |
| 🩺 **증상 (내원일지)** | 보건실 방문 기록(증상·상비약 수령·조치·변경이력) + **증상↔노출 유해인자 관련성 점검** |
| 💊 **상비약 (반출입)** | 재고·**입고(반입)**, **최근 반출(수령) 내역**, 재고 부족 경고 |
| 📚 **유해인자 카탈로그** | 분류 → 물질 → 상세로 **클릭 드릴다운**. 「유해인자별 건강장해」 편람 기반 **표적장기·급성/만성·발암성·노출기준** 정보(178종) 열람 |
| 🏃 **건강프로그램 신청·참여 현황** | 프로그램 모집·신청(정원 초과 시 자동 대기), 참여율·수료 관리 |
| 🧑‍⚕️ **건강명부** | 부서별 명부(부서 추가/수정/삭제) + 임직원 선택 시 **건강 프로필** — 유해물질 노출/특수검진·증상·약물·**건강검진(사후관리소견)·검사 수치 추이(그래프)** 통합 |

## 빠르게 실행하기

> 사전 준비: [Node.js](https://nodejs.org) 18 이상 설치

```bash
# 1) 의존성 설치 (최초 1회)
npm install

# 2) 개발 서버 실행 → 브라우저에서 http://localhost:5173 접속
npm run dev
```

처음 실행하면 예시(샘플) 데이터가 자동으로 채워져 화면을 바로 확인할 수 있습니다.

### 그 외 명령어

```bash
npm test          # 테스트 실행 (도메인 규칙 + 서비스 + 화면 스모크)
npm run typecheck # 타입 검사
npm run build     # 배포용 빌드 (dist/ 생성)
npm run preview   # 빌드 결과 미리보기
```

## 데이터 보관 & 개인정보

- 모든 데이터는 **사용 중인 브라우저의 localStorage**에만 저장됩니다(서버 전송 없음).
- 따라서 **건강정보가 외부로 나가지 않아** 프로토타입 단계에서 개인정보 보호에 유리합니다.
- 다른 PC/브라우저와 데이터가 공유되지 않으며, 브라우저 데이터를 지우면 초기화됩니다.
- 실제 다중 사용자 운영이 필요하면 저장소 구현만 교체하면 됩니다 → [docs/architecture.md](docs/architecture.md)

## 프로젝트 구조 (요약)

```
src/
  domain/          # 순수 비즈니스 규칙 (외부 의존성 0) — 검진주기·재고·정원 로직
  application/     # 유스케이스 + 포트(저장소 인터페이스)
  infrastructure/  # 포트 구현 (localStorage, 시드 데이터, 시계)
  composition/     # 의존성 조립(합성 루트)
  ui/              # React 화면
test/
  harness/         # 테스트 하네스 (인메모리 저장소, 고정 시계, 빌더)
  unit/            # 도메인 단위 테스트
  integration/     # 서비스/화면 통합 테스트
docs/              # 아키텍처·도메인 문서
```

자세한 설명은 [docs/architecture.md](docs/architecture.md)와 [CLAUDE.md](CLAUDE.md)를 참고하세요.
